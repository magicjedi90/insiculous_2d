## Open Issues

### [UX-001] No file picker dialog ‚Äî scene path is a single constant
- **File:** `src/constants.rs` (`DEFAULT_SCENE_PATH`), used by `editor_game/scene_io.rs`,
  `menu_actions.rs`, `shortcuts.rs`
- **Issue:** Open/Save As always use `scenes/scene.ron`. A real file dialog is
  Phase 2+ work; until then the path is at least defined in exactly one place.
- **Priority:** Low (tracked on PROJECT_ROADMAP Phase 2)

### [ARCH-101] `handle_create_action` matches on menu label strings
- **File:** `src/entity_ops.rs`
- **Issue:** Menu actions arrive as `&str` ("Create Sprite", ‚Ä¶) and are matched
  by string both in `editor_game/menu_actions.rs` and `entity_ops.rs`. A typo'd
  label fails silently. A `MenuAction` enum produced by the menu system would
  make this compile-checked.
- **Priority:** Low | **Effort:** Medium (touches editor menu API)

## Resolved (June 2026)

### ~~Duplicate `ComponentKind` enum + dispatch matches~~ ‚úÖ RESOLVED
- entity_ops.rs previously defined its own `ComponentKind`, `ComponentCategory`,
  `categorized_components`, `component_display_name`, `entity_has_component`,
  `available_components`, add/remove helpers, and a `clone_component!` macro;
  panel_renderer carried a `to_command_kind()` enum-to-enum converter.
- **Resolution:** All deleted. `editor::ComponentKind` (generated by the
  registry macro in `editor/src/stored_component.rs`) is the single type;
  duplication uses `editor::capture_all_components`/`restore_components`.
  Adding a component type is now one line in the editor crate's registry.
